1. DSP Matematika: Jezgreni Algoritmi

1.1 Sub Oscilator — Fazno Zaključavanje i Anti-Click
Developer Sheet definira fazu kao hardkodiranu na 0° i zahtijeva anti-click rampu ispod 2 ms. Evo egzaktne implementacije:

// Phase-locked sine oscilator s anti-click envelope
struct SubOscillator {
  double phase = 0.0;       // Uvijek 0 pri note-on
  double phaseInc = 0.0;    // = freq / sampleRate
  float  antiClickGain = 0.0f;
  int    antiClickSamples = 0;  // ceil(0.002 * sampleRate)
  int    antiClickPos = 0;

  void noteOn(double freq, double sampleRate) {
    phase = 0.0;  // HARD RESET — nikad random
    phaseInc = freq / sampleRate;
    antiClickPos = 0;
    antiClickSamples = (int)ceil(0.002 * sampleRate); // 2ms ramp
    antiClickGain = 0.0f;
  }

  float tick() {
    float sample = (float)std::sin(phase * 2.0 * M_PI);
    // Anti-click linear ramp (zamijeni s raised cosine za glatkiju krivulju)
    if (antiClickPos < antiClickSamples) {
      antiClickGain = (float)antiClickPos / antiClickSamples;
      antiClickPos++;
    } else { antiClickGain = 1.0f; }
    phase += phaseInc;
    if (phase >= 1.0) phase -= 1.0;  // Nema fmod() — skup je
    return sample * antiClickGain;
  }
};

DSP
Raised cosine ramp (½ cos krivulja) daje 6dB manje klik artefakta od linearne rampe. Za 2ms pri 44100 Hz to je 89 uzoraka — zanemariv CPU trošak.

1.2 Mid/Side Frekvencijska Skretnica (Crossover)
Ekskluzivno mono ispod 120 Hz nije trivijalna operacija. Naive pristup (hard cutoff) uvodi faznu latenciju. Preporučena implementacija je LinkwitzRiley (LR4) crossover koji garantira sumiranje bez faznog gubitka:

// Linkwitz-Riley 4. reda crossover (LR4)
// Karakteristika: savršena rekonstrukcija pri sumiranju LP+HP
class LR4Crossover {
  // Dva kaskadirana Butterworth 2. reda (svaki je 12dB/oct)
  Biquad lp1, lp2, hp1, hp2;

  void setCrossover(float freq, float sampleRate) {
    // Q = 0.7071 za Butterworth / Bessel aproksimaciju
    float omega = 2.0f * M_PI * freq / sampleRate;
    float q = 0.7071f;
    lp1.setLowpass(omega, q);  lp2.setLowpass(omega, q);
    hp1.setHighpass(omega, q); hp2.setHighpass(omega, q);
  }

  void process(float in, float& subOut, float& topOut) {
    subOut = lp2.process(lp1.process(in));  // Sve ispod 120Hz → MONO
    topOut = hp2.process(hp1.process(in));  // Sve iznad → stereo OK
  }
};

// U render petlji: prisilno monofoniziranje sub signala
float subMono = (subL + subR) * 0.5f;  // Nema width ispod crossovera

1.4 Wavefolding — Matematika i Implementacijska Napomena
Wavefolding signal koji prelazi prag zrcali natrag. Standardna jednadžba za jednu razinu foldinga:

// Jednostavni wavefolder — jednadžba za fold pri threshold T
// Ograničeno na jednu razinu (kao u analognim kolima)
float wavefold(float x, float threshold) {
  // Normalizacija na [-1, 1]
  while (x >  threshold) x = 2.0f * threshold - x;
  while (x < -threshold) x = -2.0f * threshold - x;
  return x;
}

// VAŽNO: Implementacija s while petljom je ispravna ali SKUPA za CPU.
// Za optimizaciju koristiti triangularnu wave distorziju:
float wavefolderOptimized(float x, float T) {
  x = x / (2.0f * T);   // Normalizacija
  x = x - std::floor(x + 0.5f); // Triangularna funkcija
  return 2.0f * T * std::fabs(2.0f * x) - T;  // Rekonstrukcija
}

WARN
Wavefolding generira aliasing koji raste eksponencijalno s brojem foldova. Minimalni preporučeni oversampling za fold ≥ 2: 4x OS. Za 3+ razine foldinga: 8x OS obavezan.

2.2 Legato Portamento — Implementacija
Portamento (klizanje između nota) je esencijalno za Liquid i Ragga Jungle. Nije dovoljno linearno interpolirati pitch — rezultat zvuči plastično. Preporučena krivulja je eksponencijalna:

// Eksponencijalni portamento
// Bolje od linearnog jer prati prirodnu percepciju visine tona
class Portamento {
  double currentFreq = 440.0;
  double targetFreq  = 440.0;
  double coeff = 0.999; // Ovo se izračunava iz glide time

  void setGlideTime(float ms, float sampleRate) {
    // Vremenska konstanta: signal dostiže 63% targeta u glideTime
    coeff = std::exp(-1.0 / (ms * 0.001 * sampleRate));
  }

  void setTarget(double freq) { targetFreq = freq; }

  double tick() {
    currentFreq += (targetFreq - currentFreq) * (1.0 - coeff);
    return currentFreq;
  }
};

// NAPOMENA: Glide treba bypasirati sub oscilatorsku fazu — ne resetirati je
// pri legato note-on jer bi to ubilo rollerski flow.

2.3 Voice Stealing Politika
Čak i u monofonom modu, voice stealing mora biti deterministički (ne FIFO) jer random stealing uzrokuje nepredvidiv klik:

	•	Uvijek steal: najstarija nota koja nije u attack fazi (jer attack prekida steal najglasnijeg tona)
	•	Nikad steal: nota u prvih 5 ms (anti-click ramp je još aktivan — steal bi stvorio klik)
	•	Prioritet: za Retrigger mod, sub oscilator dobiva vlastiti non-stealable slot — character engine može biti stolen

3.2 Polyphase FIR Anti-Aliasing Filter
Za svaki blok koji zahtijeva OS, upsample i downsample filteri moraju biti polyphase FIR tipa. Eliptic IIR je CPU efikasniji ali uvodi grupnu kašnjenje — problematično za attack/punch osjet:

// Pseudo-struktura za 4x OS blok
class OversampledBlock {
  // Polyphase FIR koeficijenti prekopirani iz npr. remez() dizajna
  // Cutoff: 0.45 * originalSampleRate (Nyquist zaštita)
  // Stopband: -96dB minimum (CD kvaliteta)
  static constexpr int OS_FACTOR = 4;
  PolyphaseFIR upsampleFilter;   // Interpolacijski FIR
  PolyphaseFIR downsampleFilter; // Decimacijski FIR

  float processNonlinear(float input) {
    float upsampled[OS_FACTOR];
    upsampleFilter.upsample(input, upsampled, OS_FACTOR);
    for (int i = 0; i < OS_FACTOR; ++i)
      upsampled[i] = nonlinearProcess(upsampled[i]); // wavefold/FM/clip
    return downsampleFilter.downsample(upsampled, OS_FACTOR);
  }
};

// NAPOMENA: FIR latencija = (filterLength - 1) / 2 uzoraka.
// Kompenzirati s odgovarajućim latency reportanjem u VST3 procesoru.

IMPL
HIIR biblioteka (https://github.com/AlainCoulombe/hiir) pruža besplatne, visokokvalitetne polyphase half-band filtere optimizirane za SIMD. Preporučena za produkcijsku implementaciju.

4.2 SIMD Optimizacija za Filtre
Kritični putevi koji se izvršavaju svaki uzorak (filteri, oscilatori) trebaju koristiti SIMD instrukcije. Moderna implementacija koristi ARM NEON (Apple Silicon) ili SSE/AVX (x86):

// Biquad filter — 4 paralelna glasa odjednom (SIMD SSE)
// Umjesto serijskog procesiranja 4x, procesiramo sve 4 simultano
#include <xmmintrin.h>  // SSE

struct BiquadSSE {
  __m128 b0, b1, b2, a1, a2;  // Koeficijenti za 4 filtera
  __m128 z1, z2;               // Stanje za 4 filtera

  __m128 process(__m128 x) {
    __m128 y = _mm_add_ps(_mm_mul_ps(b0, x),  z1);
    z1 = _mm_add_ps(_mm_sub_ps(_mm_mul_ps(b1, x), _mm_mul_ps(a1, y)), z2);
    z2 = _mm_sub_ps(_mm_mul_ps(b2, x), _mm_mul_ps(a2, y));
    return y;
  }
};
// Rezultat: ~3.8x brže od serijalnog na modernim CPUima

4.3 Parameter Smoothing — Denormali i Zipper Noise
Developer Sheet navodi 5–30 ms parameter smoothing ali ne daje implementacijski detalj. Koristiti one-pole IIR low-pass za smoothing:

// One-pole exponential smoother (standardni industrijski pristup)
class ParamSmoother {
  float current = 0.0f;
  float coeff   = 0.999f;

  void setSmoothTime(float ms, float sampleRate) {
    coeff = std::exp(-1.0f / (ms * 0.001f * sampleRate));
  }

  float process(float target) {
    current += (1.0f - coeff) * (target - current);
    // Anti-denormal: flush to zero blizu nule
    if (std::fabs(current) < 1e-15f) current = 0.0f;
    return current;
  }
};

// Preporučeni smoothing vremena po tipu parametra:
// Filter Cutoff: 10-20ms  |  Volume/Gain: 5ms  |  FM Ratio: 20-30ms
// LFO Rate (BPM sync): instant (bez smoothing, snap na vrijednost)

6. Napredna Filtar Topologija

6.1 State Variable Filter (SVF) — Preporučena Implementacija
Dokumenti preporučuju 12dB i 24dB filtre ali ne specificiraju topologiju. State Variable Filter (Chamberlin/TPT varijanta) je industriski standard za plugin razvoj jer je stabilan pri visokim rezonancijama i audio-rate modulaciji:

// Topology-preserving Transform (TPT) SVF — Vadim Zavalishin dizajn
// Izvor: 'The Art of VA Filter Design' (besplatno dostupno)
struct TPT_SVF {
  float ic1eq = 0, ic2eq = 0;  // Integrator stanje

  // Koeficijenti — izračunaj jednom pri promjeni cutoff/Q
  float g, k, a1, a2, a3;

  void setParams(float cutoffHz, float Q, float sampleRate) {
    g = std::tan(M_PI * cutoffHz / sampleRate);
    k = 1.0f / Q;
    a1 = 1.0f / (1.0f + g * (g + k));
    a2 = g * a1;
    a3 = g * a2;
  }

  // Vraća sve izlaze simultano (LP, BP, HP)
  void process(float in, float& LP, float& BP, float& HP) {
    float v3 = in - ic2eq;
    float v1 = a1 * ic1eq + a2 * v3;
    float v2 = ic2eq + a2 * ic1eq + a3 * v3;
    ic1eq = 2.0f * v1 - ic1eq;
    ic2eq = 2.0f * v2 - ic2eq;
    LP = v2;  BP = v1;  HP = in - k * v1 - v2;
  }
};
// Kaskadiranje 2x TPT_SVF daje LR4 (24dB/oct) bez faznih artefakata

IMPL
TPT SVF podržava audio-rate cutoff modulaciju bez nestabilnosti (za razliku od standardnih biquad filtera). Kritično za LFO → filter cutoff modulaciju u Roller basovima.

7. Sub-Harmonik Generator — Nedokumentirana Tehnika

7.1 Oktavni Sub-Harmonik za Povećanje Dubine
Tekuća dokumentacija ne pokriva sub-harmonik generator koji mnogi top DnB producenti koriste za dodatnu dubinu bez pojačavanja temeljnog tona. Princip je generiranje signala na frekvenciji točno jedne oktave ispod note:

// Sub-harmonik generator (1 oktava ispod, 50% phase offset)
// Generira čisti sinus na f/2 — maximalna kompatibilnost s PA sustavima
class SubHarmonicGenerator {
  double subPhase = 0.0;
  float level = 0.0f;  // Default: 0% (isključeno), max: 30%

  void noteOn(double fundFreq, double sampleRate) {
    subPhase = 0.0;  // Fazno zaključano kao i main sub
    subPhaseInc = (fundFreq * 0.5) / sampleRate;  // f/2 = oktava niže
  }

  float tick() {
    float sub = (float)std::sin(subPhase * 2.0 * M_PI) * level;
    subPhase += subPhaseInc;
    if (subPhase >= 1.0) subPhase -= 1.0;
    return sub;
  }
};
// LIMIT: level max 30% — više uzrokuje maskiranje kick drum-a ispod 40Hz
// Preporučena frekvencijska zona za sub-harmonik: 20–45 Hz

INFO
Sub-harmonik na f/2 dodaje energiju ispod 50Hz što je čujno samo na klupskim PA sustavima i DJ monitorima. Korisna opcija za Jungle i Dark DnB stilove gdje sub mora biti 'fizički'. Treba biti skrivena kontrola, ne makro.

8.2 DC Blocker Implementacija

// Jednostavni 1-polni DC blocker (visokopropusni IIR)
// R = 0.995 daje cutoff ~22Hz pri 44100Hz — dovoljno za DC uklanjanje
class DCBlocker {
  float xPrev = 0.0f, yPrev = 0.0f;
  float R = 0.995f;  // Povećaj za niže cutoff (sporije reagiranje)

  float process(float x) {
    float y = x - xPrev + R * yPrev;
    xPrev = x;  yPrev = y;
    return y;
  }
};
// NAPOMENA: DC blocker uvodi minimalnu grupnu kašnjenje (< 1 uzorak)
// ali ne uzrokuje percepcijsku promjenu u sub-basu.

9.2 VST3 State Serijalizacija
VST3 saveState() / loadState() pozivi trebaju serijalizirati sve parametre kao XML ili kompaktni binarni blob. Kritična pravila:

	•	Verzija oznaka: Svaki state mora uključivati plugin verziju broj — za migraciju starih preseta
	•	Unknown parametri: Nepoznate vrijednosti pri load-u trebaju biti tiho ignorirane (ne crash) — za forward kompatibilnost
	•	Randomize funkcija: Treba biti implementirana ali ograničena na DnB-validni prostor — ne stvara generičke sintetske zvukove
	•	Init preset: Hard-coded 'Init' preset vraća sve na default vrijednosti iz Developer Sheet tablice

10. UI/UX Smjernice za Restriktivni Instrument

10.1 Vizualni Jezik Ograničenja
Ključni UX izazov: korisnik mora OSJEĆATI slobodu dok je zapravo zatvoren u DnB-validan prostor. Principi koji to postižu:

	•	Skriveni ograničeni raspon — Kontrola izgleda kao da ide od 0–100%, ali interno mapira na npr. 10ms–300ms za release. Korisnik ne vidi limit.
	•	Inkrementalni unlock — Osnovna sučelja (Performance mode) prikazuju 4 makroa. Advanced mode otkriva modulacijsku matricu. Expert mode prikazuje sve interne parametre.
	•	Vizualni feedback distorzijskog karaktera — Osciloskop koji prikazuje sub val (uvijek čist sinus) i character val (bogat, distorziran) u real-time.
	•	Spektralni analizator — Real-time FFT koji prikazuje sub-topline ispod 120Hz u drukčijoj boji od character spectral sadržaja.
	•	Makro animacija — Kada se Neuro-Formant vrti, UI prikazuje kretanje notch filtera na spektralnom prikazu.

10.2 Style Mode Tranzicija
Prebacivanje između Style Modova (Tech/Neuro/Liquid/Jump Up) ne smije biti instant klik već glatka interpolacija svih parametara kroz 50–100 ms. Ovo sprječava audibilne artefakte pri live sviranju:

// Style Mode morph — interpolacija između 2 seta parametara
// duration: 50–100ms (korisnik može podesiti u postavkama)
class StyleModeMorph {
  float morphProgress = 1.0f;  // 0.0 = od, 1.0 = do
  float morphRate = 0.0f;      // Izračunato iz duration
  ParamSet fromParams, toParams;

  void triggerMorph(ParamSet from, ParamSet to, float durationMs, float sampleRate) {
    fromParams = from;  toParams = to;
    morphProgress = 0.0f;
    morphRate = 1.0f / (durationMs * 0.001f * sampleRate);
  }

  ParamSet tick() {
    morphProgress = std::min(1.0f, morphProgress + morphRate);
    // Koristiti smoothstep za S-krivulju (perceptivno prirodnija)
    float t = morphProgress * morphProgress * (3.0f - 2.0f * morphProgress);
    return lerp(fromParams, toParams, t);
  }
};

10.3 Vizualni Identitet — Tehničke Preporuke
Za DnB VST koji cilja tech/neuro/dark podžanrove, vizualni jezik treba biti:

	•	Tamna tema obavezna — nikakav bijeli ili sivi background (light theme asocira na amaterske plugin-e)
	•	Minimalni broj fontova — max 2 fonta; monospaced za parametre, sans-serif za labele
	•	Boja upozorenja — Crvena/narančasta samo za DSP safety limitere (vizualni signal da je korisnik na rubu)
	•	Animirani VU meteri za sub i character kanal — prikazuju energiju u real-time
	•	Osciloskop view opcionalan ali preporučen za expert mode — vizualna povratna informacija o faznom zaključavanju
