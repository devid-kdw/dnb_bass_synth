1. DSP Matematika: Jezgreni Algoritmi





1.1 Sub Oscilator вҖ” Fazno ZakljuДҚavanje i Anti-Click

Developer Sheet definira fazu kao hardkodiranu na 0В° i zahtijeva anti-click rampu ispod 2 ms. Evo egzaktne implementacije:



// Phase-locked sine oscilator s anti-click envelope

struct SubOscillator {

  double phase = 0.0;       // Uvijek 0 pri note-on

  double phaseInc = 0.0;    // = freq / sampleRate

  float  antiClickGain = 0.0f;

  int    antiClickSamples = 0;  // ceil(0.002 * sampleRate)

  int    antiClickPos = 0;



  void noteOn(double freq, double sampleRate) {

    phase = 0.0;  // HARD RESET вҖ” nikad random

    phaseInc = freq / sampleRate;

    antiClickPos = 0;

    antiClickSamples = (int)ceil(0.002 * sampleRate); // 2ms ramp

    antiClickGain = 0.0f;

  }



  float tick() {

    float sample = (float)std::sin(phase * 2.0 * M_PI);

    // Anti-click linear ramp (zamijeni s raised cosine za glatkiju krivulju)

    if (antiClickPos < antiClickSamples) {

      antiClickGain = (float)antiClickPos / antiClickSamples;

      antiClickPos++;

    } else { antiClickGain = 1.0f; }

    phase += phaseInc;

    if (phase >= 1.0) phase -= 1.0;  // Nema fmod() вҖ” skup je

    return sample * antiClickGain;

  }

};



DSP
Raised cosine ramp (ВҪ cos krivulja) daje 6dB manje klik artefakta od linearne rampe. Za 2ms pri 44100 Hz to je 89 uzoraka вҖ” zanemariv CPU troЕЎak.




1.2 Mid/Side Frekvencijska Skretnica (Crossover)

Ekskluzivno mono ispod 120 Hz nije trivijalna operacija. Naive pristup (hard cutoff) uvodi faznu latenciju. PreporuДҚena implementacija je LinkwitzRiley (LR4) crossover koji garantira sumiranje bez faznog gubitka:



// Linkwitz-Riley 4. reda crossover (LR4)

// Karakteristika: savrЕЎena rekonstrukcija pri sumiranju LP+HP

class LR4Crossover {

  // Dva kaskadirana Butterworth 2. reda (svaki je 12dB/oct)

  Biquad lp1, lp2, hp1, hp2;



  void setCrossover(float freq, float sampleRate) {

    // Q = 0.7071 za Butterworth / Bessel aproksimaciju

    float omega = 2.0f * M_PI * freq / sampleRate;

    float q = 0.7071f;

    lp1.setLowpass(omega, q);  lp2.setLowpass(omega, q);

    hp1.setHighpass(omega, q); hp2.setHighpass(omega, q);

  }



  void process(float in, float& subOut, float& topOut) {

    subOut = lp2.process(lp1.process(in));  // Sve ispod 120Hz вҶ’ MONO

    topOut = hp2.process(hp1.process(in));  // Sve iznad вҶ’ stereo OK

  }

};

1.4 Wavefolding вҖ” Matematika i Implementacijska Napomena

Wavefolding signal koji prelazi prag zrcali natrag. Standardna jednadЕҫba za jednu razinu foldinga:



// Jednostavni wavefolder вҖ” jednadЕҫba za fold pri threshold T

// OgraniДҚeno na jednu razinu (kao u analognim kolima)

float wavefold(float x, float threshold) {

  // Normalizacija na [-1, 1]

  while (x >  threshold) x = 2.0f * threshold - x;

  while (x < -threshold) x = -2.0f * threshold - x;

  return x;

}



// VAЕҪNO: Implementacija s while petljom je ispravna ali SKUPA za CPU.

// Za optimizaciju koristiti triangularnu wave distorziju:

float wavefolderOptimized(float x, float T) {

  x = x / (2.0f * T);   // Normalizacija

  x = x - std::floor(x + 0.5f); // Triangularna funkcija

  return 2.0f * T * std::fabs(2.0f * x) - T;  // Rekonstrukcija

}



WARN
Wavefolding generira aliasing koji raste eksponencijalno s brojem foldova. Minimalni preporuДҚeni oversampling za fold вүҘ 2: 4x OS. Za 3+ razine foldinga: 8x OS obavezan.

3.2 Polyphase FIR Anti-Aliasing Filter

Za svaki blok koji zahtijeva OS, upsample i downsample filteri moraju biti polyphase FIR tipa. Eliptic IIR je CPU efikasniji ali uvodi grupnu kaЕЎnjenje вҖ” problematiДҚno za attack/punch osjet:



// Pseudo-struktura za 4x OS blok

class OversampledBlock {

  // Polyphase FIR koeficijenti prekopirani iz npr. remez() dizajna

  // Cutoff: 0.45 * originalSampleRate (Nyquist zaЕЎtita)

  // Stopband: -96dB minimum (CD kvaliteta)

  static constexpr int OS_FACTOR = 4;

  PolyphaseFIR upsampleFilter;   // Interpolacijski FIR

  PolyphaseFIR downsampleFilter; // Decimacijski FIR



  float processNonlinear(float input) {

    float upsampled[OS_FACTOR];

    upsampleFilter.upsample(input, upsampled, OS_FACTOR);

    for (int i = 0; i < OS_FACTOR; ++i)

      upsampled[i] = nonlinearProcess(upsampled[i]); // wavefold/FM/clip

    return downsampleFilter.downsample(upsampled, OS_FACTOR);

  }

};



// NAPOMENA: FIR latencija = (filterLength - 1) / 2 uzoraka.

// Kompenzirati s odgovarajuДҮim latency reportanjem u VST3 procesoru.

4.2 SIMD Optimizacija za Filtre

KritiДҚni putevi koji se izvrЕЎavaju svaki uzorak (filteri, oscilatori) trebaju koristiti SIMD instrukcije. Moderna implementacija koristi ARM NEON (Apple Silicon) ili SSE/AVX (x86):



// Biquad filter вҖ” 4 paralelna glasa odjednom (SIMD SSE)

// Umjesto serijskog procesiranja 4x, procesiramo sve 4 simultano

#include <xmmintrin.h>  // SSE



struct BiquadSSE {

  __m128 b0, b1, b2, a1, a2;  // Koeficijenti za 4 filtera

  __m128 z1, z2;               // Stanje za 4 filtera



  __m128 process(__m128 x) {

    __m128 y = _mm_add_ps(_mm_mul_ps(b0, x),  z1);

    z1 = _mm_add_ps(_mm_sub_ps(_mm_mul_ps(b1, x), _mm_mul_ps(a1, y)), z2);

    z2 = _mm_sub_ps(_mm_mul_ps(b2, x), _mm_mul_ps(a2, y));

    return y;

  }

};

// Rezultat: ~3.8x brЕҫe od serijalnog na modernim CPUima




4.3 Parameter Smoothing вҖ” Denormali i Zipper Noise

Developer Sheet navodi 5вҖ“30 ms parameter smoothing ali ne daje implementacijski detalj. Koristiti one-pole IIR low-pass za smoothing:



// One-pole exponential smoother (standardni industrijski pristup)

class ParamSmoother {

  float current = 0.0f;

  float coeff   = 0.999f;



  void setSmoothTime(float ms, float sampleRate) {

    coeff = std::exp(-1.0f / (ms * 0.001f * sampleRate));

  }



  float process(float target) {

    current += (1.0f - coeff) * (target - current);

    // Anti-denormal: flush to zero blizu nule

    if (std::fabs(current) < 1e-15f) current = 0.0f;

    return current;

  }

};

6. Napredna Filtar Topologija





6.1 State Variable Filter (SVF) вҖ” PreporuДҚena Implementacija

Dokumenti preporuДҚuju 12dB i 24dB filtre ali ne specificiraju topologiju. State Variable Filter (Chamberlin/TPT varijanta) je industriski standard za plugin razvoj jer je stabilan pri visokim rezonancijama i audio-rate modulaciji:



// Topology-preserving Transform (TPT) SVF вҖ” Vadim Zavalishin dizajn

// Izvor: &apos;The Art of VA Filter Design&apos; (besplatno dostupno)

struct TPT_SVF {

  float ic1eq = 0, ic2eq = 0;  // Integrator stanje



  // Koeficijenti вҖ” izraДҚunaj jednom pri promjeni cutoff/Q

  float g, k, a1, a2, a3;



  void setParams(float cutoffHz, float Q, float sampleRate) {

    g = std::tan(M_PI * cutoffHz / sampleRate);

    k = 1.0f / Q;

    a1 = 1.0f / (1.0f + g * (g + k));

    a2 = g * a1;

    a3 = g * a2;

  }



  // VraДҮa sve izlaze simultano (LP, BP, HP)

  void process(float in, float& LP, float& BP, float& HP) {

    float v3 = in - ic2eq;

    float v1 = a1 * ic1eq + a2 * v3;

    float v2 = ic2eq + a2 * ic1eq + a3 * v3;

    ic1eq = 2.0f * v1 - ic1eq;

    ic2eq = 2.0f * v2 - ic2eq;

    LP = v2;  BP = v1;  HP = in - k * v1 - v2;

  }

};

// Kaskadiranje 2x TPT_SVF daje LR4 (24dB/oct) bez faznih artefakata



IMPL
TPT SVF podrЕҫava audio-rate cutoff modulaciju bez nestabilnosti (za razliku od standardnih biquad filtera). KritiДҚno za LFO вҶ’ filter cutoff modulaciju u Roller basovima.

7. Sub-Harmonik Generator вҖ” Nedokumentirana Tehnika





7.1 Oktavni Sub-Harmonik za PoveДҮanje Dubine

TekuДҮa dokumentacija ne pokriva sub-harmonik generator koji mnogi top DnB producenti koriste za dodatnu dubinu bez pojaДҚavanja temeljnog tona. Princip je generiranje signala na frekvenciji toДҚno jedne oktave ispod note:



// Sub-harmonik generator (1 oktava ispod, 50% phase offset)

// Generira ДҚisti sinus na f/2 вҖ” maximalna kompatibilnost s PA sustavima

class SubHarmonicGenerator {

  double subPhase = 0.0;

  float level = 0.0f;  // Default: 0% (iskljuДҚeno), max: 30%



  void noteOn(double fundFreq, double sampleRate) {

    subPhase = 0.0;  // Fazno zakljuДҚano kao i main sub

    subPhaseInc = (fundFreq * 0.5) / sampleRate;  // f/2 = oktava niЕҫe

  }



  float tick() {

    float sub = (float)std::sin(subPhase * 2.0 * M_PI) * level;

    subPhase += subPhaseInc;

    if (subPhase >= 1.0) subPhase -= 1.0;

    return sub;

  }

};

// LIMIT: level max 30% вҖ” viЕЎe uzrokuje maskiranje kick drum-a ispod 40Hz

// PreporuДҚena frekvencijska zona za sub-harmonik: 20вҖ“45 Hz



INFO
Sub-harmonik na f/2 dodaje energiju ispod 50Hz ЕЎto je ДҚujno samo na klupskim PA sustavima i DJ monitorima. Korisna opcija za Jungle i Dark DnB stilove gdje sub mora biti &apos;fiziДҚki&apos;. Treba biti skrivena kontrola, ne makro.

// Jednostavni 1-polni DC blocker (visokopropusni IIR)

// R = 0.995 daje cutoff ~22Hz pri 44100Hz вҖ” dovoljno za DC uklanjanje

class DCBlocker {

  float xPrev = 0.0f, yPrev = 0.0f;

  float R = 0.995f;  // PoveДҮaj za niЕҫe cutoff (sporije reagiranje)



  float process(float x) {

    float y = x - xPrev + R * yPrev;

    xPrev = x;  yPrev = y;

    return y;

  }

};

10.2 Style Mode Tranzicija

Prebacivanje izmeД‘u Style Modova (Tech/Neuro/Liquid/Jump Up) ne smije biti instant klik veДҮ glatka interpolacija svih parametara kroz 50вҖ“100 ms. Ovo sprjeДҚava audibilne artefakte pri live sviranju:



// Style Mode morph вҖ” interpolacija izmeД‘u 2 seta parametara

// duration: 50вҖ“100ms (korisnik moЕҫe podesiti u postavkama)

class StyleModeMorph {

  float morphProgress = 1.0f;  // 0.0 = od, 1.0 = do

  float morphRate = 0.0f;      // IzraДҚunato iz duration

  ParamSet fromParams, toParams;



  void triggerMorph(ParamSet from, ParamSet to, float durationMs, float sampleRate) {

    fromParams = from;  toParams = to;

    morphProgress = 0.0f;

    morphRate = 1.0f / (durationMs * 0.001f * sampleRate);

  }



  ParamSet tick() {

    morphProgress = std::min(1.0f, morphProgress + morphRate);

    // Koristiti smoothstep za S-krivulju (perceptivno prirodnija)

    float t = morphProgress * morphProgress * (3.0f - 2.0f * morphProgress);

    return lerp(fromParams, toParams, t);

  }

};

8. NajvaЕҫnije Arhitekturne Odluke вҖ” ADR SaЕҫetak




PreporuДҚuje se docs/adr/ folder za Architecture Decision Records. Ovo su prve ДҚetiri odluke koje treba dokumentirati:



ADR
ADR-001
JUCE 8 odabran naspram iPlug2 i DPF вҖ” JUCE pruЕҫa jedini kompletan paket (parameter state + GUI + preset + MIDI + multi-target) bez potrebe za zasebnom integracijom svake komponente. iPlug2 zahtijeva viЕЎe ruДҚnog wiring-a za state menadЕҫment. DPF nema JUCE-ekvivalentni GUI sistem.

ADR
ADR-002
Per-module oversampling (HIIR) naspram globalnog oversamplinga вҖ” Globalni OS poveДҮava CPU load za sve module ukljuДҚujuДҮi filtre koji ne generiraju aliasing. HIIR polyphase FIR primjenjuje OS samo na FM, Wavefold i WT вҖ” procijenjeni CPU gain: 30-50% naspram globalnog 4x OS.

ADR
ADR-003
domain/ sloj kao jedini ulaz u dsp/ вҖ” Svaka alternativa (direktno ДҚitanje JUCE parametara u DSP modulu) duplicira constraint logiku, poveДҮava rizik od regresije i oteЕҫava testiranje Еҫanrovskih pravila.

ADR
ADR-004
CLAP.h dodan pasivno u MVP вҖ” Zero-cost ukljuДҚivanje, ali aktivacija odgoД‘ena za V2. SprjeДҚava kasniji refactor jer CLAP.h ne interferira s JUCE VST3 targetom.
