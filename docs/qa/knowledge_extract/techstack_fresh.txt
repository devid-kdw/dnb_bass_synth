
DNB BASS VST3
Tech Stack, Arhitektura & Folder Struktura
Analiziran, dopunjen i finaliziran prijedlog
Verzija 1.0  ·  Kompatibilno s: Research V2 · Developer Spec · DSP Supplement



Ovaj dokument analizira predloženi tech stack i folder strukturu, donosi konkretne dopune i ispravke, te isporučuje finaliziranu verziju spremnu za postavljanje repozitorija. Svaka promjena naspram originalnog prijedloga eksplicitno je označena i obrazložena. Tamo gdje je original točan, zadržan je bez izmjena.

1. Analiza i Ocjena Originalnog Prijedloga

1.1 Što je dobro (zadržati bez izmjena)
Sljedeće odluke iz originalnog prijedloga su ispravne i dovoljno argumentirane:

	•	C++20 kao jedini runtime jezik — ispravan izbor, DSP ekosustav živi u C++
	•	JUCE + CMake workflow — jedina razumna opcija za VST3 na macOS/Windows bez reimplementacije plugin-a od nule
	•	Python samo za tooling (QA skripte, batch render, analiza) — ne u runtime-u
	•	git submodules za dependencies (JUCE, Catch2, nlohmann/json) — reproducibilan build
	•	Odvajanje dsp/ od domain/ sloja — ključna arhitektonska odluka za ovaj projekt
	•	Catch2 za unit testove — lagan, header-friendly, dobar C++ test framework
	•	spdlog samo u editor/init threadu, nikad u audio callbacku — RT-safe pristup
	•	4 build konfiguracije (Debug, RelWithDebInfo, Release, ASan/UBSan) — profesionalan setup
	•	CI od prvog dana — audio plugin build regresije su skupi kad ih se kasno uhvati

5. CMakeLists.txt Skica

Minimalna CMake struktura koja pokriva sve navedene dependencies i targets:

cmake_minimum_required(VERSION 3.24)
project(DnBBassSynth VERSION 1.0.0)
set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# ── Submodules ───────────────────────────────────────────────────────
add_subdirectory(extern/juce)
add_subdirectory(extern/catch2)
add_subdirectory(extern/spdlog)
add_subdirectory(extern/benchmark)
add_subdirectory(extern/hiir)     # header-only — samo include

# ── Compiler warnings (Warnings-as-errors u CI) ──────────────────────
include(cmake/CompilerWarnings.cmake)
include(cmake/Sanitizers.cmake)

# ── Plugin target ────────────────────────────────────────────────────
juce_add_plugin(DnBBassSynth
    PLUGIN_MANUFACTURER_CODE  Dnbs
    PLUGIN_CODE               Dbs1
    FORMATS                   VST3           # Dodati AU/CLAP u V2
    PRODUCT_NAME              "DnB Bass Synth"
    IS_SYNTH                  TRUE
    NEEDS_MIDI_INPUT          TRUE
)

target_sources(DnBBassSynth PRIVATE
    src/app/PluginProcessor.cpp
    src/app/PluginEditor.cpp
    # ... ostali .cpp fajlovi
)

target_include_directories(DnBBassSynth PRIVATE
    src/
    extern/hiir/
    extern/json/single_include/
)

target_link_libraries(DnBBassSynth PRIVATE
    juce::juce_audio_processors
    juce::juce_audio_utils
    juce::juce_dsp
    juce::juce_gui_basics
    spdlog::spdlog
    juce::juce_recommended_config_flags
    juce::juce_recommended_warning_flags
)

# ── Tests ────────────────────────────────────────────────────────────
enable_testing()
add_subdirectory(tests)

# ── Benchmarks ───────────────────────────────────────────────────────
add_subdirectory(bench)

8. Najvažnije Arhitekturne Odluke — ADR Sažetak

Preporučuje se docs/adr/ folder za Architecture Decision Records. Ovo su prve četiri odluke koje treba dokumentirati:

ADR
ADR-001
JUCE 8 odabran naspram iPlug2 i DPF — JUCE pruža jedini kompletan paket (parameter state + GUI + preset + MIDI + multi-target) bez potrebe za zasebnom integracijom svake komponente. iPlug2 zahtijeva više ručnog wiring-a za state menadžment. DPF nema JUCE-ekvivalentni GUI sistem.
ADR
ADR-002
Per-module oversampling (HIIR) naspram globalnog oversamplinga — Globalni OS povećava CPU load za sve module uključujući filtre koji ne generiraju aliasing. HIIR polyphase FIR primjenjuje OS samo na FM, Wavefold i WT — procijenjeni CPU gain: 30-50% naspram globalnog 4x OS.
ADR
ADR-003
domain/ sloj kao jedini ulaz u dsp/ — Svaka alternativa (direktno čitanje JUCE parametara u DSP modulu) duplicira constraint logiku, povećava rizik od regresije i otežava testiranje žanrovskih pravila.
ADR
ADR-004
CLAP.h dodan pasivno u MVP — Zero-cost uključivanje, ali aktivacija odgođena za V2. Sprječava kasniji refactor jer CLAP.h ne interferira s JUCE VST3 targetom.

9. Quick-Start za Novi Developer

Redosljed koraka za postavljanje dev okruženja na macOS od nule:

# 1. Xcode Command Line Tools (jednom)
xcode-select --install

# 2. brew dependencies
brew install cmake llvm ninja

# 3. Clone repo s submodulima (one-shot)
git clone --recurse-submodules https://github.com/yourorg/dnb-bass-synth.git
cd dnb-bass-synth

# Alternativa ako je repo već cloniran:
bash tools/scripts/init_submodules.sh

# 4. Configure + Build (Debug)
cmake -B build/debug -DCMAKE_BUILD_TYPE=Debug -G Ninja
cmake --build build/debug --parallel

# 5. Run unit tests
cd build/debug && ctest --output-on-failure

# 6. Build Release (za install u DAW)
cmake -B build/release -DCMAKE_BUILD_TYPE=Release -G Ninja
cmake --build build/release --parallel
# VST3 output: build/release/DnBBassSynth_artefacts/VST3/

# 7. Format check
bash tools/scripts/format.sh --check

ℹ
init_submodules.sh
Skripta mora pokriti: git submodule update --init --recursive. Ovo je najčešće zaboravljen korak koji uzrokuje 'missing header' build errore kod novih developera.
